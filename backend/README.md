# backend

## リポジトリ構成

あとで清書する

- cmd
  - エントリーポイントフォルダ。batchとかのエントリーポイントもここに来る予定。
- env
  - アプリ内で使用する環境変数管理フォルダ。適切な粒度があれば、もっと大きい粒度でどこかとまとめてもいいかも。
- external
  - discordなど外部ドメインに依存する処理はここ。todoとして、テスト作成時mockにしやすいようにinterfaceにしたい。
- handler
  - 各エンドポイントのエントリーポイントになるハンドラー関数の定義場所。こいつが色々なinterfaceに依存する。
- helper
  - handlerで使う便利関数。utilsだとスコープが大きく感じてこの命名にしているが変わらないかも…？
- infra
  - DBやRedisなどの外部ストレージを操作するクライアントの定義はここ。S3もここ？
- middleware
  - APIのミドルウェア定義。トークン生成はココじゃないかもと思いつつ、切り出す場所がイマイチでひとまず置いてある
- repository
  - DBの操作を抽象化するフォルダ。こいつもdbをinterfaceとしてmockを噛んだテストを書きたい。
- router
  - エンドポイントを定義するフォルダ。prodでフロントエンドの生成物を配信する処理もここにある。
- schema
  - このバックエンドのリクエスト、レスポンス、DTOなどの複数レイヤーで使用される型を定義するフォルダ。依存はしない想定。
- store
  - Redisの操作を抽象化するフォルダ。mockを噛んだテストを書きたい。
