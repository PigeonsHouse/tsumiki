# 進捗投稿サービス

## 概要

作品の進捗を追える投稿サービス
1作品の制作過程をツリー(1直線)で閲覧できる

## タイトル候補

- Tsumiki
- Tsumiki+
- Tsumiki Hub
- Tsumiki Station
- Tsumiki Base
- Tsumiki Gallery
- Tsumiki Box

## 要件定義

### 機能要件

- (must) ログイン
  - Discord認証
  - 「ものづくりの森」ユーザのみ投稿可能
- (must) 作品投稿に関して
  - データは、作品・進捗の二種類
  - 作品には、進捗の連なりが複数本生えて、同一アカウントで複数本作ることも可能
  - 作品のデータ(作品・Work)
    - タイトル
    - 説明文
    - サムネイル
  - 進捗ツリーのルートデータ(積み木・Tsumiki)
    - 作者
    - タイトル
    - 制作時間
      - みること連携したら入力されるイメージ
    - サムネイル(必須)
    - リアクション
      - いくつかテンプレートを用意して押せる
    - コメント
    - 拍手
      - いっぱい押せるいいねみたいなやつ
      - max 100? 10? 50?
    - 公開範囲
      - 全体公開・限定公開(要ログイン)のみ
  - 進捗のデータ(ブロック・Block)
    - 画像、音声などの進捗が見れるメディア
      - なくても
    - 進捗の概要(説明文的な)
      - なくても
      - メディアか進捗、いずれかは必須とする
    - 進捗パーセンテージ(0~100)(ツリーが進んで数字が減っても良い)
      - 100パーセントになったら、続きの進捗は投稿できない
      - 新規機能追加とかなら新しいツリーを、ちょっとバグ修正とかなら完成進捗を99%とかに戻してもらう
    - 調子
      - パワプロの5段階みたいな
    - リアクション
      - いくつかテンプレートを用意して押せる
    - コメント
- (want) 検索機能
- (want) 通知機能
  - リアクション、いいね、拍手、コメントなどが来たときに通知を飛ばす
- (want) アカウントのSNS等紐付け機能
- (want) イベント情報
  - 鯖内投稿祭や、外部の作品投稿祭、コミケなど
- (should) みるこのーとと連携して、制作時間が表示される
  - あくまでみるこが見た時間
- (should) 実績機能
  - はじめての投稿、○回進捗投稿した、はじめて積み木が完成した、積み木が100件投稿された、久しぶりに進捗が生まれた、etc…

### 非機能要件

- 技術選定
  - フロントエンド
    - React
  - バックエンド
    - Go
  - インフラ
    - VPS
      - それほどメディアストレージの圧迫は起きづらいと判断
      - メディアストレージの移行が必要になったらそれだけR2とかにお引越し？
    - ローカル+TailScaleのfunnelで公開？
      - セキュリティ的なリスクが怖い
    - VPS？
      - メディアストレージの圧迫が悩ましい
    - AWS？
      - 料金が青天井で怖い&外資なのでちょっとお高い
  - コンテナ
    - AWSなら料金を抑えるためEC2を使いたい
    - それ以外なら環境構築の簡単化のためコンテナを使いたい
- DB、メディアのバックアップを取る
