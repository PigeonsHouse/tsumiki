# フロントエンド ページ設計

ヘッダーにはログイン、ログアウト、投稿、マイページなどの導線が配置される

- TOP
  - 最近動きがあった積み木
  - リアクションの多い積み木
  - 新しい積み木
  - …などなど、サマリ的な表示
  - 積み木一覧、作品一覧への導線
- 積み木一覧
- 積み木詳細
  - ブロックが表示される
  - 自分の積み木の場合、続きを積む導線もある
- 作品一覧
- 作品詳細
  - 関連積み木一覧も出る
- マイページ
  - こっちもTOPと同じくサマリっぽいページ
- マイページ/積み木一覧
  - 新規積み木作成、積み木へのブロック追加の導線もある
- マイページ/関わった作品一覧
  - 新規作品投稿の導線もある
- ブロック投稿
  - 新しく積み上げる(積み木作成も同時に行う)
  - 続きから積み上げる(既存の積み木に新しいブロックを積む)
  - 積み木の編集、既存のブロックの編集も同じUIの想定
- 作品投稿
  - 既存の作品の編集も同じUIの想定
